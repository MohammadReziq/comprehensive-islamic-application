================================================================================
  1. فلسفة تصميم الأدوار
================================================================================

القاعدة الذهبية:
═══════════════════════════════════════════════════════
  "كل ما صعدنا بالهرم الإداري ← قلّت المهام اليومية
   وزادت المسؤوليات الإشرافية"
═══════════════════════════════════════════════════════

  ولي الأمر  → يومياً (يتابع حضور أطفاله كل يوم)
  المشرف     → يومياً (يسجل حضور الأطفال 5 مرات)
  الإمام      → أسبوعياً (يراقب ويدير ويراجع)
  السوبر أدمن → شهرياً (ينظم مسابقات ويراقب النظام)

لماذا هذا الفصل مهم:
  • المشرف متطوع — لو حمّلناه مهام إدارية سيتركها
  • الإمام مشغول بالخطب والدروس — لا يريد يراجع كل حضور
  • ولي الأمر يريد راحة البال — لا يريد يسجل بنفسه
  • السوبر أدمن يدير عشرات المساجد — لا يتدخل بالتفاصيل

المبدأ النفسي:
  "كل شخص يشعر بالإنجاز حين تكون مهامه واضحة ومحدودة.
   الغموض والتداخل يسبب الإحباط والتراجع."

  ← لذلك: كل دور له حدود واضحة — لا تداخل.


================================================================================
  2. السوبر أدمن — مدير النظام
================================================================================

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  2.1 فلسفة الدور
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  السوبر أدمن هو "وزارة الأوقاف" الرقمية.
  لا يدخل المسجد يومياً — بل يراقب من بعيد ويتدخل فقط عند الحاجة.

  غايته: ضمان أن النظام يعمل بنزاهة وعدالة لجميع المساجد.

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  2.2 الصلاحيات الكاملة
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  ■ إدارة المساجد:
    ✅ الموافقة على طلبات المساجد الجديدة
    ✅ رفض طلبات المساجد مع سبب
    ✅ تعليق مسجد معتمد (إيقاف مؤقت)
    ✅ إعادة تفعيل مسجد موقوف
    ✅ تغيير إمام المسجد (نقل ملكية)
    ✅ عرض كل المساجد مع إحصائياتها

  ■ إدارة المستخدمين:
    ✅ إنشاء حساب إمام جديد (يدوياً)
    ✅ عرض كل المستخدمين (فلترة بالدور)
    ✅ تغيير دور مستخدم (ترقية/تخفيض)
    ✅ حظر مستخدم مخالف

  ■ إدارة المسابقات:
    ✅ إنشاء مسابقة عامة (multi-mosque)
    ✅ اختيار المساجد المشاركة
    ✅ تحديد نقاط الصلوات للمسابقة العامة
    ✅ تفعيل/إيقاف المسابقة العامة
    ✅ تمديد المسابقة العامة
    ✅ إضافة/إزالة مسجد من المسابقة العامة
    ✅ عرض الترتيب العام + ترتيب كل مسجد

  ■ إدارة النظام:
    ✅ تعديل إعدادات النظام (نافذة الحضور)
    ✅ عرض إحصائيات النظام (مساجد، مستخدمين، أطفال، حضور)
    ✅ عرض سجل التدقيق (Audit Log)

  ■ ما لا يفعله:
    ❌ لا يسجل حضور — أبداً وتحت أي ظرف
    ❌ لا يعالج طلبات تصحيح — ليست مسؤوليته
    ❌ لا يرسل ملاحظات لأولياء الأمور
    ❌ لا يتدخل بإعدادات مسجد بعينه
    ❌ لا ينشئ مسابقة محلية لمسجد (الإمام يفعلها)
    ❌ لا يضيف/يزيل مشرفين

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  2.3 حدود الدور
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  السوبر أدمن لا يتدخل في الشؤون اليومية للمسجد.

  مثال: لو مشرف سجّل حضور خاطئ:
    ❌ السوبر أدمن لا يصحّح  → الإمام أو المشرف يصحّح
    ❌ السوبر أدمن لا يتواصل مع ولي الأمر

  مثال: لو إمام يريد تغيير نقاط صلاة:
    ❌ السوبر أدمن لا يتدخل → الإمام يعدّل من لوحته
    ⚠ إلا في المسابقة العامة — النقاط يحددها السوبر أدمن

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  2.4 سيناريوهات الطوارئ
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  ■ إمام حُذف أو حُظر:
    → السوبر أدمن يعيّن إمام بديل عبر changeImam()
    → المسجد يبقى بكل بياناته

  ■ مسجد بدون إمام (مسجد يتيم):
    → يظهر للسوبر أدمن بحالة "يتيم"
    → يعيّن إمام جديد

  ■ شكوى رسمية من ولي أمر:
    → يراجع Audit Log
    → يتخذ قرار (تعليق مشرف عبر تعليق المسجد + تواصل مع الإمام)

  ■ تلاعب بالنقاط:
    → يراجع سجلات الحضور
    → يعلّق المسجد حتى المراجعة


================================================================================
  3. الإمام — مدير المسجد
================================================================================

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  3.1 فلسفة الدور
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  الإمام هو "مدير المدرسة" — يدير ولا ينفّذ.
  يُعيّن المشرفين ويراقبهم ويتدخل فقط عند الضرورة.

  واقعياً: إمام المسجد رجل مشغول — خطب جمعة، دروس، إمامة صلوات.
  لا يملك وقتاً لتسجيل حضور 50 طفل يومياً.

  ← لذلك: مهامه إدارية بالأساس.

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  3.2 الصلاحيات الكاملة
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  ■ إدارة المسجد:
    ✅ إنشاء المسجد (اسم، عنوان، موقع GPS)
    ✅ تعديل إعدادات المسجد
    ✅ تعديل نقاط الصلوات (prayer_config)
    ✅ عرض/مشاركة كود الدعوة

  ■ إدارة المشرفين:
    ✅ إنشاء حساب مشرف (اسم + إيميل + كلمة سر)
    ✅ عرض قائمة المشرفين
    ✅ عرض بيانات المشرف مجدداً (للمشاركة)
    ✅ إزالة مشرف
    ✅ عرض أداء المشرف (عدد تحضيرات، عدد طلبات عالجها)

  ■ إدارة المسابقات المحلية:
    ✅ إنشاء مسابقة (اسم، تواريخ)
    ✅ تفعيل/إيقاف المسابقة
    ✅ تعديل المسابقة (الاسم، تمديد التاريخ)
    ✅ عرض الترتيب (Leaderboard)
    ✅ مشاركة النتائج

  ■ المسابقة العامة (عند المشاركة):
    ✅ عرض المسابقة العامة (للقراءة فقط)
    ✅ عرض ترتيب مسجده في المسابقة العامة
    ✅ عرض الترتيب العام بين المساجد
    ❌ لا يعدّل ولا يوقف ولا يحذف المسابقة العامة

  ■ الحضور (طوارئ فقط):
    ⚠ يستطيع تسجيل حضور — لكن كآلية طوارئ
    ✅ تسجيل حضور لأي تاريخ ماضٍ (بلا حدود زمنية)
    ✅ إلغاء حضور خاطئ (override مطلق)
    ⚠ عند فتح شاشة التحضير: "أنت تسجل كإمام — هل المشرفون غير متاحين؟"

  ■ طلبات التصحيح:
    ✅ عرض كل الطلبات (المعلقة + المقبولة + المرفوضة)
    ✅ التدخل: إلغاء قبول خاطئ، أو قبول طلب مرفوض
    ✅ يستلم Escalation بعد 7 أيام من طلب معلق

  ■ الإعلانات:
    ✅ إنشاء إعلان للمسجد
    ✅ تعديل/حذف إعلان

  ■ الإحصائيات:
    ✅ إحصائيات المسجد (حضور يومي/أسبوعي/شهري)
    ✅ أفضل/أسوأ طفل
    ✅ نسبة الحضور حسب الصلاة

  ■ ما لا يفعله:
    ❌ لا ينشئ مسابقة عامة (السوبر أدمن فقط)
    ❌ لا يدير مساجد أخرى
    ❌ لا يرى بيانات مساجد أخرى
    ❌ لا يدير مستخدمين خارج مسجده

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  3.3 لماذا الإمام لا يسجل حضور بشكل يومي؟
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  من واقع إدارة المساجد (30 سنة خبرة):

  1. الإمام في الصلاة — لا يستطيع يمسك الجوال وقت الأذان
  2. الإمام يقرأ القرآن بعد الصلاة — وقته الأثمن
  3. المشرف موجود خصيصاً لهذه المهمة
  4. لو الإمام يسجل → المشرف يتراخى ("الإمام بعمل شغلي")

  الاستثناء الوحيد: كل المشرفين غابوا.
  ← الإمام يسجل بنفسه
  ← النظام يسجّل في Audit Log أن الإمام سجّل (للمراجعة لاحقاً)


================================================================================
  4. المشرف — المنفّذ الميداني
================================================================================

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  4.1 فلسفة الدور
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  المشرف هو "المعلم في الفصل" — ينفّذ الخطة اليومية.
  يفتح التطبيق ← يسجل الحضور ← يغلق. هذا كل شيء.

  واقعياً: المشرف شاب متطوع (15-25 سنة غالباً).
  يتطوع حباً في المسجد — لا راتب ولا مقابل.
  لو أرهقناه بمهام إدارية → سيترك التطوع.

  المبدأ النفسي:
  "المتطوع يستمر حين يشعر أن مهمته واضحة وقصيرة ومُقدَّرة."
  ← لذلك: أبسط دور ممكن + كلمة شكر من الإمام.

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  4.2 الصلاحيات الكاملة
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  ■ تسجيل الحضور:
    ✅ تسجيل حضور (QR / رقم محلي / بحث بالاسم)
    ✅ تسجيل حضور لآخر 3 أيام (قابل للتعديل من الإمام)
    ✅ إلغاء حضور سجّله بنفسه (خلال 24 ساعة)
    ❌ لا يسجل حضور مستقبلي (ممنوع على الجميع)
    ❌ لا يعدّل حضور أقدم من 3 أيام (للقراءة فقط)

  ■ طلبات التصحيح:
    ✅ عرض الطلبات المعلقة لمسجده
    ✅ قبول طلب (يعدّل الحضور تلقائياً)
    ✅ رفض طلب (مع سبب إجباري)
    ❌ لا يرى طلبات سابقة لتاريخ انضمامه

  ■ الملاحظات:
    ✅ إرسال ملاحظة لولي أمر طفل
    ✅ عرض الملاحظات التي أرسلها

  ■ المسابقات (قراءة فقط):
    ✅ عرض المسابقة النشطة (المحلية والعامة)
    ✅ عرض الترتيب
    ❌ لا ينشئ ولا يعدّل ولا يوقف مسابقة

  ■ الإعلانات:
    ✅ قراءة إعلانات المسجد
    ❌ لا ينشئ ولا يعدّل إعلانات

  ■ ملخص يومي:
    ✅ عدد الأطفال الذين حضّرهم اليوم (حسب الصلاة)

  ■ ما لا يفعله:
    ❌ لا مهام إدارية إطلاقاً
    ❌ لا يدير المشرفين الآخرين
    ❌ لا يصل لإعدادات المسجد
    ❌ لا يرى Audit Log
    ❌ لا يرى إحصائيات شاملة
    ❌ لا ينشئ حسابات


================================================================================
  5. ولي الأمر — المتابع
================================================================================

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  5.1 فلسفة الدور
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  ولي الأمر هو "المتابع عن بُعد" — يراقب ويُبلّغ.
  لا يملك أي صلاحية تنفيذية.

  واقعياً: أب يعمل طوال اليوم — يريد يفتح التطبيق ويرى:
  "هل ابني صلّى الفجر اليوم؟" ← نعم ✅ أو لا ❌

  لو الإجابة خاطئة ← يرسل طلب تصحيح.
  هذا كل ما يحتاجه.

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  5.2 الصلاحيات الكاملة
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  ■ إدارة الأطفال:
    ✅ إضافة طفل (اسم، عمر، جنس)
    ✅ تعديل بيانات طفل
    ✅ ربط طفل بمسجد (عبر كود 6 أحرف)
    ✅ فك ربط طفل من مسجد

  ■ متابعة الحضور:
    ✅ عرض حضور اليوم لكل أطفاله (Realtime)
    ✅ عرض من سجّل الحضور (اسم المشرف)
    ✅ عرض سجل حضور مفصل (يومي/أسبوعي/شهري)

  ■ طلبات التصحيح:
    ✅ إرسال طلب تصحيح (اختيار الصلاة + التاريخ + ملاحظة)
    ✅ عرض حالة طلباته (معلق/مقبول/مرفوض)
    ✅ حد أقصى 3 طلبات معلقة بنفس الوقت (Anti-Spam)

  ■ المسابقات:
    ✅ عرض المسابقة النشطة (بانر على الرئيسية)
    ✅ عرض ترتيب أطفاله في المسابقة
    ✅ عرض الترتيب العام (كل الأطفال)

  ■ الإشعارات:
    ✅ استلام إشعار عند الرد على طلب تصحيح
    ✅ قراءة ملاحظات من المشرف
    ✅ قراءة إعلانات المسجد

  ■ Gamification:
    ✅ عرض نقاط كل طفل
    ✅ عرض مستوى الطفل (بذرة → أمير)
    ✅ عرض شارات الطفل
    ✅ عرض بطاقة الطفل (QR + رقم محلي)

  ■ ما لا يفعله:
    ❌ لا يسجل حضور بنفسه — أبداً
    ❌ لا يعدّل أي سجل مباشرة
    ❌ لا يرى أطفال الآخرين
    ❌ لا مهام إدارية
    ❌ لا يتواصل مع المشرف مباشرة (فقط عبر طلبات التصحيح)


================================================================================
  6. الطفل — مستقبلاً
================================================================================

  حالياً: الطفل ليس مستخدم في النظام — بياناته يديرها ولي الأمر.

  مستقبلاً (بعد الإطلاق):
    ✅ عرض بطاقته (QR Code) من جهازه
    ✅ عرض نقاطه وشاراته ومستواه
    ✅ عرض حضوره اليومي
    ❌ لا يسجل حضور بنفسه
    ❌ لا يعدّل أي شيء

  الربط: ولي الأمر يربط جهاز ابنه عبر كود مؤقت.
  (Migration 028 child_account.sql موجودة)

================================================================================
  8. سيناريوهات الطوارئ
================================================================================

  السيناريو 1: كل المشرفين غابوا أسبوع
  ────────────────────────────────────────
    → الإمام يسجل الحضور بنفسه (بلا حدود زمنية)
    → يظهر في Audit: "سُجّل بواسطة الإمام"
    → بعد عودة المشرفين: الإمام يتوقف عن التسجيل

  السيناريو 2: إمام سافر شهر
  ────────────────────────────────
    → المشرفون يستمرون بالتسجيل (لا يتأثرون)
    → طلبات التصحيح تُعالج من المشرفين
    → لو بقي طلب 7 أيام: Escalation للإمام (عند عودته)

  السيناريو 3: مشرف أساء استخدام صلاحياته
  ──────────────────────────────────────────
    → الإمام يرى في Audit Log الأنماط الغريبة
    → الإمام يزيل المشرف فوراً
    → سجلات الحضور التي سجّلها تبقى (للمراجعة)
    → لو الضرر كبير: الإمام يتواصل مع السوبر أدمن

  السيناريو 4: سقوط النظام (Supabase down)
  ────────────────────────────────────────────
    → مؤقتاً: لا أحد يستطيع التسجيل
    → مستقبلاً: Offline Mode (مؤجل)
    → الإمام يسجل بأثر رجعي بعد عودة النظام

  9. العلاقات الإشرافية
================================================================================

  ┌───────────────────────────────────────────────┐
  │              السوبر أدمن                      │
  │         يراقب النظام + مسابقات عامة            │
  │                    │                           │
  │         ┌──────────┴──────────┐                │
  │         ▼                     ▼                │
  │     الإمام (أ)            الإمام (ب)           │
  │    يدير مسجد أ           يدير مسجد ب           │
  │         │                     │                │
  │    ┌────┴────┐           ┌────┴────┐           │
  │    ▼         ▼           ▼         ▼           │
  │ مشرف 1   مشرف 2      مشرف 3   مشرف 4         │
  │ ينفّذ    ينفّذ       ينفّذ    ينفّذ           │
  │    │         │           │         │           │
  │    ▼         ▼           ▼         ▼           │
  │ أولياء أمور ← يتابعون أطفالهم                  │
  └───────────────────────────────────────────────┘

  قواعد الإشراف:
  • السوبر أدمن لا يتجاوز الإمام — يتواصل معه أولاً
  • الإمام لا يتجاوز المشرف في المهام اليومية
  • المشرف لا يتدخل بشؤون مشرف آخر
  • ولي الأمر يتواصل فقط عبر طلبات التصحيح (لا رسائل مباشرة)

  التصعيد (Escalation):
  طلب تصحيح → مشرف (7 أيام) → إمام (تلقائياً)
  شكوى أمنية → إمام → سوبر أدمن (خارج النظام)
