================================================================================
  4. خطة الإطلاق التجريبي
================================================================================

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  4.1 المسجد الأول — كيف نختاره
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  معايير الاختيار:
  1. إمام متحمس ومتعاون (يفهم التكنولوجيا)
  2. مشرف واحد على الأقل (متطوع ملتزم)
  3. 10-30 طفل (عدد كافٍ لكنه لا يربكنا)
  4. إنترنت متاح في المسجد
  5. قرب جغرافي منك (تقدر تروح تتابع)

  الأفضل: مسجد تعرف إمامه شخصياً.

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  4.2 خطوات الإطلاق التجريبي
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  اليوم 0:
  - اجتمع مع الإمام والمشرف (شرح التطبيق — 30 دقيقة)
  - ثبّت التطبيق على أجهزتهم
  - أنشئ حساب الإمام (أنت كسوبر أدمن)
  - الإمام ينشئ المسجد ← تعتمده من لوحتك
  - الإمام يُنشئ حساب المشرف

  الأسبوع 1 (تجريبي — بدون مسابقة):
  - المشرف يسجل حضور الأطفال يومياً
  - أولياء الأمور يثبتون التطبيق ويربطون أطفالهم
  - راقب: هل الحضور يُسجَّل؟ هل النقاط تُحسب؟
  - اجمع ملاحظات (مباشرة + واتساب)

  الأسبوع 2 (تجريبي — مع مسابقة محلية):
  - الإمام ينشئ مسابقة "تحدي الأسبوع"
  - راقب: الترتيب، حماس الأطفال، ردود أولياء الأمور
  - اجمع ملاحظات

  الأسبوع 3 (تقييم):
  - اجتمع مع الإمام والمشرف
  - ما نجح؟ ما لم ينجح؟ ما يحتاج تحسين؟
  - قائمة بالـ bugs والتحسينات

  الأسبوع 4 (إصلاحات + توسيع):
  - أصلح الـ bugs الحرجة
  - لو كل شيء جيد ← أضف مسجد ثاني
  - لو مشاكل كبيرة ← أصلح أولاً

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  4.3 ماذا نراقب (KPIs)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  ┌──────────────────────────────────┬──────────────────────────────┐
  │ المؤشر                         │ الهدف                        │
  ├──────────────────────────────────┼──────────────────────────────┤
  │ عدد الأطفال المسجلين            │ 10+ في أول أسبوع             │
  │ نسبة أولياء الأمور الفعّالين    │ 50%+ يفتحون التطبيق يومياً  │
  │ نسبة الحضور المُسجَّل           │ 80%+ من الأطفال كل يوم      │
  │ عدد طلبات التصحيح               │ < 5/أسبوع (يعني التسجيل دقيق)│
  │ مدة المشرف في التطبيق           │ < 5 دقائق لكل صلاة           │
  │ أخطاء تقنية (crashes)          │ 0 في أول أسبوع               │
  │ رضا الإمام (1-10)              │ 7+                            │
  │ رضا أولياء الأمور (1-10)        │ 7+                            │
  └──────────────────────────────────┴──────────────────────────────┘

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  4.4 متى ننتقل من تجريبي لعام
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  شروط الانتقال:
  1. ✅ 3 مساجد تعمل بنجاح لمدة أسبوعين
  2. ✅ لا أخطاء حرجة (crashes = 0)
  3. ✅ رضا المستخدمين > 7/10
  4. ✅ الحضور يُسجَّل بدقة (طلبات تصحيح < 3/أسبوع/مسجد)
  5. ✅ المشرف لا يشتكي من صعوبة الاستخدام

  ← عندها: نُنزل على المتاجر (App Store + Google Play)


================================================================================
  5. المخاطر وخطة التعامل
================================================================================

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  5.1 أعلى 5 مخاطر
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  المخاطرة 1: المشرف لا يسجل حضور (تراخي)
  ────────────────────────────────────────
    الاحتمال: متوسط (المشرف متطوع — قد يملّ)

    الحل:
    - الإمام يرى في Audit: "اليوم لم يُسجَّل حضور"
    - تحذير تلقائي: "مسجدك بلا حضور منذ X أيام"
    - الإمام يتواصل مع المشرف
    - الإمام يسجل بنفسه (طوارئ)
    - حل طويل المدى: تعيين أكثر من مشرف

  المخاطرة 2: أولياء الأمور لا يثبتون التطبيق
  ────────────────────────────────────────────
    الاحتمال: عالٍ (الناس كسولة عادةً)

    الحل:
    - النظام يعمل بدون ولي الأمر (المشرف يسجل والإمام يرى)
    - حافز: "شاهد ترتيب ابنك في المسابقة!" ← يحفز التثبيت
    - الإمام يروّج في خطبة الجمعة
    - كرت صغير للطفل "حمّل تطبيق صلاتي حياتي"

  المخاطرة 3: مشاكل أوقات الصلاة (خطأ بالحساب)
  ────────────────────────────────────────────────
    الاحتمال: منخفض (لكن أثره كبير)

    الحل:
    - الإمام يحدد موقع المسجد (lat/lng) ← أوقات دقيقة
    - نافذة حضور سخية (24 ساعة افتراضي)
    - لو خطأ: المشرف يلاحظ ← الإمام يعدّل الموقع

  المخاطرة 4: تلاعب بالحضور (مشرف يسجل أطفال لم يحضروا)
  ────────────────────────────────────────────────────────
    الاحتمال: منخفض (لكن يُفقد المصداقية)

    الحل:
    - Audit Log: كل حضور يُسجَّل مع recorded_by + timestamp
    - الإمام يراجع Audit أسبوعياً
    - أنماط غريبة (50 طفل في دقيقة) ← تحذير تلقائي
    - مستقبلاً: QR متجدد (يمنع التسجيل بدون وجود الطفل)

  المخاطرة 5: فشل Supabase (downtime)
  ────────────────────────────────────
    الاحتمال: منخفض جداً (Supabase 99.9% uptime)

    الحل:
    - الحضور لا يُفقد (يُسجَّل بأثر رجعي عند العودة)
    - الإمام يمكنه تسجيل آخر 72 ساعة بدون مشكلة
    - مستقبلاً: Offline Mode

  6. معايير النجاح (KPIs) — بعد 3 أشهر
================================================================================

  ┌──────────────────────────────────┬──────────────────────────────┐
  │ المؤشر                         │ الهدف                        │
  ├──────────────────────────────────┼──────────────────────────────┤
  │ عدد المساجد المسجلة             │ 10+                          │
  │ عدد المساجد النشطة              │ 5+ (حضور يومي)               │
  │ إجمالي الأطفال                  │ 200+                         │
  │ مسابقة عامة ناجحة               │ 1 على الأقل (3+ مساجد)       │
  │ تقييم المتجر                    │ 4.0+ نجوم                    │
  │ retention (بعد شهر)             │ 40%+ من أولياء الأمور         │
  │ NPS (هل تنصح صديقك)            │ 7+/10                        │
  └──────────────────────────────────┴──────────────────────────────┘

  لو حققنا هذه الأرقام ← المشروع ناجح ← نوسّع.
  لو لم نحققها ← نراجع ← نعدّل ← نعيد المحاولة.

  ╔═══════════════════════════════════════════════════════════════╗
  ║  التطبيق ليس مجرد أداة حضور.                                ║
  ║  هو منصة تحفيز ديني = مسابقة + Realtime + تحليلات.          ║
  ║  هذا USP فريد في السوق العربية — لا مثيل له.               ║
  ║                                                               ║
  ║  "صلاتي حياتي" اسم ملهم — خلّيه يكون واقعاً ملهماً.        ║
